Из функции в класс
Мы переработали функцию RunCalculatorCycle. Теперь в ней нет логики работы калькулятора. Всю логику работы калькулятора нужно реализовать в отдельном классе.
Что нужно сделать
Создайте класс Calculator, который реализует всю функциональность калькулятора из финальной задачи прошлого спринта.
Класс должен хранить текущее число и число в памяти и содержать следующие методы:
void Set(Number n). Заменяет текущее число (результат) на число n.
Number GetNumber() const. Возвращает текущий результат вычислений калькулятора. В только что сконструированном калькуляторе этот метод возвращает 0.
void Add(Number n). Прибавляет число n к текущему результату внутри калькулятора.
void Sub(Number n). Вычитает число n из текущего результата.
void Div(Number n). Делит текущий результат на n.
void Mul(Number n). Умножает текущий результат на n.
void Pow(Number n). Возводит текущий результат в степень n.
void Save(). Сохраняет текущий результат в ячейку памяти калькулятора.
void Load(). Загружает число из памяти калькулятора в текущий результат.
bool HasMem() const. Возвращает true, если ячейка памяти непустая.
std::string GetNumberRepr() const. Возвращает текущее число, преобразуя его в std::string функцией std::to_string.
Для реализации метода HasMem добавьте флаг состояния ячейки памяти в поля класса. Его нужно возвести при вызове метода Save.
В файле main.cpp вы найдёте код для тестирования класса. Используйте примеры ниже для проверки реализации функций.
Ограничения
Реализуйте все методы из условия.
Разместите определения методов в файле calculator.cpp.
Не меняйте содержимое файла main.cpp.
