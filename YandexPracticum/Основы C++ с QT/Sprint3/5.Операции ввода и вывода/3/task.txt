Форматирование ввода и вывода — полезная опция, но перегрузки могут пригодиться и в более интересных задачах. Мы предлагаем вам дописать игру «Кузнечик», в которой переопределённые операции ввода-вывода играют ключевую роль. 
Эта игра позволяет вам управлять кузнечиком, охотящимся за мушками. Кузнечик перемещается по прямой, прыгая вперёд или назад. Как только он ловит мушку, следующая появляется на случайной свободной позиции.
Ниже приведён пример хода игры.

k------o--        // Исходное положение кузнечика и мушки.
5                 // Перемещаем кузнечика на пять позиций вперёд.
-----k-o--
2
Current score: 1  // Кузнечик поймал первую мушку.
o------k--        
-7                // Перемещаем кузнечика на семь позиций назад.
Current score: 2  // Кузнечик поймал вторую мушку.
k-o-------
2
Current score: 3  // Игра длится до трёх очков.
You win! 
У перемещения кузнечика есть несколько особенностей.
В начале игры он находится на позиции под номером 0.
Игровое поле ограничено справа и слева. Оказавшись в самой правой клеточке, кузнечик может двигаться только влево. При попытке переместить его правее его позиция не меняется. С левой стороной это работает аналогично.
Для управления кузнечиком в коде реализован класс Grasshopper, содержащий информацию о его текущем положении. Также для него перегружены операции >> и <<. В этой задаче перегрузки работают необычным образом: в качестве левого операнда они принимают не объект потока, а объект класса Grasshopper (но по-прежнему возвращают его через return).
Операция >> позволяет переместить кузнечика вправо на указанное количество клеточек. Можно использовать как положительные, так и отрицательные числа.

Grasshopper grasshopper;
// Начальное положение:  k---o-----
grasshopper >> 3;       // ---ko-----
grasshopper >> 15;      // ----o----k
grasshopper >> 2;       // ----o----k
grasshopper >> -1;      // ----o---k- 
Аналогично операция << позволяет передвинуть кузнечика влево.

Grasshopper grasshopper; 
// Начальное положение: k-o-------
grasshopper << 2;      // k-o-------
grasshopper << -7;     // --o----k--
grasshopper << 3;      // --o-k-----   
Перегрузка операций работает таким образом, что можно использовать каскадирование.

Grasshopper grasshopper; 
// Начальное положение: k---o-----
grasshopper >> 2 >> 5; // ----o--k--
grasshopper << 1 << 1; // ----ok----
grasshopper >> 3 << 1; // ----o--k-- 
Что нужно сделать
Напишите реализацию перегрузок ввода и вывода для класса Grasshopper в файле grasshopper_game.h. Номер позиции кузнечика хранится в приватной переменной pos_grasshopper_, которая может принимать значения в диапазоне [0, board_size_ - 1], где board_size_ — ещё одна переменная класса, равная размеру игрового поля.
Чтобы «зажать» позицию кузнечика в заданном диапазоне, используйте функцию std::clamp(value, min, max) из библиотеки algorithm. Она возвращает value, если она принадлежит интервалу [min, max]: значение min, если value выходит за границы интервала слева, и max, если справа.

#include <algorithm>

std::clamp(12, 10, 20) // Результат: 12.
std::clamp(5, 10, 20)  // Результат: 10.
std::clamp(21, 10, 20) // Результат: 20. 
В заготовке вам дано:
Класс Grasshopper в файле grasshopper_game.h, где реализованы методы для управления кузнечиком и мушкой.
Класс Game в файле game.h с реализацией логики игры. Игра продолжается, пока количество набранных очков (переменная score) не будет равным заданному (переменная winning_score, по умолчанию равная 3). Очки считаются по количеству пойманных мушек.
Для решения задачи вам не обязательно анализировать класс Game.

Ограничения
В перегрузках ввода-вывода для проверки границ поля используйте функцию std::clamp.
Помните, что функции, определяемые в заголовочных файлах, должны быть встроенными (inline).
