На многих сайтах при регистрации требуется ввести адрес электронной почты. Там же производится его валидация — проверка того, что введённые данные действительно похожи на электронный ящик. Это делается прежде всего для удобства пользователя — чтобы избежать ошибок типа ввода имени вместо адреса почты.
Обычно к email-адресам предъявляются такие требования:
В строке должен быть ровно один символ @ (официальное название — «коммерческое at», неофициальное — «собака»), и после этого символа должна быть хотя бы одна точка.
Между @ и первой после неё точкой должен быть хотя бы один символ.
После последней точки должен быть символ.
Строка может состоять только из латинских символов (больших и маленьких), точек, «собаки», знаков _, -, + и цифр.
Вот все допустимые символы, перечисленные в одном литерале: "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789._-+@".
Что нужно сделать
Напишите программу, которая по введённой из потока строке проверит, является ли эта строка адресом электронной почты или нет. Если строка является email-адресом, то выведите строку Корректный email, иначе — Некорректный email.
Алгоритм решения
Для решения этой задачи вам будет необходимо последовательно проверить все условия:
Проще всего начать с поиска символа @. Если в строке нет этого символа, значит, адрес некорректный. Иначе переходим к анализу второй части адреса.
Если во второй части есть ещё один символ @, значит, адрес некорректный.
Ищем символ точки во второй части адреса. Если точек нет или точка следует сразу за символом @ (то есть позиция точки равна позиции «собаки» + 1), значит, адрес некорректный.
Теперь нужно проверить, что адрес не заканчивается точкой. Для этого ищем позицию последней точки с помощью метода rfind.
Осталось проверить валидность символов. Если в строке есть недопустимые символы, значит, адрес некорректный.
Если все эти условия выполняются, значит, адрес корректный.
Вы можете воспользоваться дополнительной переменной is_correct типа bool, в которую записывается false, если какая-то из проверок не прошла. Не забудьте присвоить переменной изначальное значение true. В конце выведите ответ в зависимости от значения переменной.
Как будет тестироваться код
Вывод вашей программы будет сравниваться с эталонным.
Примеры
Пример 1
Входные данные
hello@world.ru 
Результат
Корректный email 
Объяснение
Введённая строка удовлетворяет условиям корректности email-адреса: содержит @ и . в нужном порядке. Ответ — Корректный email.
Пример 2
Входные данные
Hello, world! 
Результат
Некорректный email 
Объяснение
Введённая строка не удовлетворяет условиям корректности email-адреса: не содержит ни @, ни .. Ответ — Некорректный email.
