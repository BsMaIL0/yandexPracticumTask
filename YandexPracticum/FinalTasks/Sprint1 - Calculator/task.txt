Следующий этап калькулятора — выделение в нём функций. Вы можете вводить собственные функции по желанию, но две должны быть обязательно:
Функция ReadNumber, которая будет читать число и выводить сообщение об ошибке в std::cerr в случае неудачи.
Функция RunCalculatorCycle, которая будет содержать всю логику калькулятора.
В функции main должен располагаться только вызов RunCalculatorCycle.
Калькулятор должен вместить всю функциональность и все команды предыдущей версии, но сверх того мы добавим в калькулятор память: ячейку, способную сохранять одно число.
Что нужно сделать
В калькулятор добавляются две команды:
s — сохранить число в ячейку памяти.
l — загрузить число из памяти.
Если число не сохранялось, но встретилась команда l, выводится сообщение об ошибке: Error: Memory is empty. И калькулятор завершает работу.
Вместо типа double используйте псевдоним Number, создав его командой using Number = double;.
Используйте функции ReadNumber и RunCalculatorCycle со следующими сигнатурами:

// Возвращает true, если удалось прочитать число
// и сохранить его в Number.
bool ReadNumber(Number& result);

// Возвращает true, если работа завершилась штатно 
// командой q. Если из-за ошибки, возвращает false.
bool RunCalculatorCycle(); 
Напомним команды калькулятора из предыдущей версии:
+ <число> — сложить число в калькуляторе с прочитанным числом;
аналогично — операции -, *, / и ** — возвести в степень;
= — вывести текущее число в калькуляторе;
: <число> — явно задать число в калькуляторе;
c — очистить (установить число 0);
q — выйти из программы.
Если после команды +, -, *, /, ** или : не удалось прочитать число, программа выводит в std::cerr сообщение Error: Numeric operand expected и завершает работу.
Это же сообщение нужно вывести, если программа не смогла прочитать начальное число.
Если программа прочитала неизвестную команду, она выводит в cerr сообщение: Error: Unknown token <команда>. И тоже завершает работу. В этом сообщении нужно вывести прочитанное слово на месте плейсхолдера <команда>.
Ограничения
У вас должны быть функции ReadNumber и RunCalculatorCycle с заданными сигнатурами.
Тип double должен использоваться только в using.
Определения этих функций должны располагаться в файле calculator.cpp. Определение функции main должно быть в файле main.cpp.
Чтобы пройти ревью, соблюдайте руководство по стилю (стайлгайд).
Как будет тестироваться код
Тесты проверят, что у вас есть функции ReadNumber и RunCalculatorCycle, которые имеют нужные сигнатуры и работают правильно.
Также будет проверена работа калькулятора.
Функция main() будет игнорироваться при проверке.
